# DevChat — AI 编程产品套件 · 产品设计文档

## 1. 产品概述

### 1.1 产品定位
DevChat 是一套面向开发者的 AI 编程产品，由移动端主项目 App 与官网 App 组成。主项目 App 让用户通过对话方式驱动 AI 对 GitHub 仓库进行代码开发，在手机上即可完成代码修改、审查和提交；官网 App 承担产品介绍、下载引导、更新公告和文档入口。

### 1.2 核心价值
| 价值点 | 描述 |
|--------|------|
| **随时随地编程** | 通勤、会议间隙即可处理代码任务 |
| **零门槛使用** | 自然语言对话，无需记命令 |
| **安全可控** | 所有变更需用户确认后才提交 |
| **AI 上下文感知** | 自动生成项目快照，AI 持续理解项目全貌 |
| **清晰触达** | 官网提供产品说明、下载入口、更新公告和支持文档 |

### 1.3 目标用户
- 需要移动端轻量开发的全栈/后端开发者
- 想用 AI 辅助快速修改代码的独立开发者
- 需要在手机上做 Code Review 和热修复的团队负责人
- 首次了解 DevChat、需要下载入口或查阅文档的潜在用户

## 2. 功能模块

### 2.1 功能全景图

```
DevChat
├── 官网 App
│   ├── 产品介绍
│   ├── 下载与安装引导
│   ├── 更新公告
│   └── 文档与支持入口
│
└── 主项目 App
    ├── 账户与配置
    │   ├── AI 模型配置（URL + API Key）
    │   ├── GitHub 账号绑定（OAuth）
    │   └── 偏好设置（主题/语言/默认分支）
    ├── 项目管理
    │   ├── GitHub Repo 列表
    │   ├── 项目快照生成与缓存
    │   ├── 项目切换
    │   └── 最近项目
    ├── AI 对话开发
    │   ├── 对话界面（类 Chat）
    │   ├── AI 返回 Diff 展示
    │   ├── 修改文件列表
    │   ├── 多轮对话上下文
    │   └── 历史会话记录
    └── 代码提交
        ├── Diff 确认
        ├── Commit Message 编辑
        ├── 选择性提交（勾选文件）
        └── Push 到远程
```

### 2.2 核心功能详述

#### F1: AI 模型配置
- 支持 OpenAI 兼容 API 格式（覆盖 Claude / GPT / DeepSeek / 各种代理）
- 用户输入：`Base URL` + `API Key` + `Model Name`
- 支持多套配置，可切换
- 连接测试按钮，验证配置可用性

#### F2: GitHub 集成
- OAuth 2.0 授权登录
- 获取用户所有 Repo（public + private）
- 支持搜索/筛选仓库
- 显示仓库基本信息（语言、Star、最近更新时间）

#### F3: 项目快照（Project Snapshot）
- **首次进入项目**时，AI 自动分析项目结构：
  - 目录树（忽略 node_modules 等）
  - 关键文件识别（入口文件、配置文件、核心模块）
  - 技术栈检测（语言/框架/依赖）
  - 模块关系摘要
- 生成结构化 JSON/YAML 快照，存储在本地
- **后续进入**直接加载快照作为 AI 上下文，秒级启动
- 支持手动刷新快照

#### F4: 对话式开发
- Chat UI，支持 Markdown 渲染
- 用户用自然语言描述需求
- AI 基于项目快照 + 对话上下文，返回：
  - 代码修改方案（Diff 格式）
  - 修改说明
  - 影响范围分析
- 支持追问、修正、撤销上一步修改
- 支持引用特定文件进行讨论

#### F5: Diff 展示与审查
- 类 GitHub Diff 的展示界面
- 红绿标记增删行
- 文件级别的折叠/展开
- 修改文件列表侧边栏
- 支持逐文件勾选（选择性提交）

#### F6: 提交与推送
- 自动生成 Commit Message（可编辑）
- 一键 Commit + Push
- 支持选择目标分支
- 提交历史记录

#### F7: 官网 App
- 使用独立官网承载产品定位、核心功能、下载入口、更新公告和支持文档入口
- 面向搜索引擎和社交分享优化，支持基础 SEO、Open Graph 信息和响应式展示
- 与主项目 App 共享品牌视觉、基础设计 token 和可复用 UI 组件
- 不承载用户密钥、GitHub Token 或项目数据，仅处理公开内容和必要的访问统计

## 4. 数据模型

### 4.1 核心实体

```
User
├── ai_configs: [{ name, base_url, api_key, model }]
├── github_token: string
└── preferences: { theme, language }

Project
├── repo_id: string
├── repo_name: string
├── snapshot: ProjectSnapshot
├── last_accessed: timestamp
└── sessions: [Session]

Session (对话会话)
├── project_id: string
├── messages: [Message]
├── pending_changes: [FileDiff]
├── status: active | committed
└── created_at: timestamp

ProjectSnapshot
├── directory_tree: object
├── tech_stack: { language, framework, dependencies }
├── key_files: [{ path, role, summary }]
├── module_map: object
└── generated_at: timestamp
```

## 5. 非功能性需求

| 维度 | 要求 |
|------|------|
| **性能** | 项目快照生成 < 30s，对话响应支持流式输出 |
| **安全** | API Key 本地加密存储，GitHub Token 使用 Keychain/Keystore |
| **离线** | 项目快照、历史会话离线可查 |
| **体积** | 安装包 < 30MB |
| **官网体验** | 首屏快速加载，移动端与桌面端都能清晰完成下载和文档访问 |
