> 当前同步日期：2026-05-12
> 本文档描述当前已实现产品形态，并标注仍在计划中的能力。

## 1. 产品概述

### 1.1 产品定位

DevChat 是一套面向开发者的 AI 编程产品，由主项目 App 与官网 App 组成。

- 主项目 App：基于 Tauri 2 + React + TypeScript 的移动优先编程工作台，当前在桌面开发环境中验证。用户选择 GitHub 仓库后，DevChat 读取真实项目上下文，通过 AI 对话生成可审查 Diff，并在用户确认后提交到远程分支。
- 官网 App：基于 Next.js App Router 的产品展示站，承担产品介绍、功能说明、下载入口、更新公告和文档入口。

### 1.2 核心价值

| 价值点 | 描述 |
|--------|------|
| **对话式开发** | 用户用自然语言描述需求，AI 基于真实项目快照和对话历史生成修改建议 |
| **审查后提交** | AI 输出被解析为统一 Diff，用户逐文件选择后再 Commit & Push |
| **真实项目上下文** | 从 GitHub 读取 repo、branch、tree 和文件内容，生成本地项目快照 |
| **安全凭据边界** | API Key 与 GitHub Token 写入系统安全存储，普通配置只保存 secret ref 和非敏感元数据 |
| **离线可回看** | 最近项目、项目快照和历史会话保存在本地，已提交会话进入只读状态 |
| **清晰触达** | 官网提供产品说明、功能入口、下载状态和支持文档 |

### 1.3 目标用户

- 想用 AI 快速理解和修改 GitHub 仓库的开发者
- 需要在轻量设备上处理小型代码变更、热修复或 Review 的开发者
- 需要可审查、可选择、可追踪 AI 代码变更的独立开发者和团队负责人
- 首次了解 DevChat、需要下载入口或配置说明的潜在用户

### 2.2 核心功能详述

#### F1: AI 模型配置

- 支持两类 Provider：`Anthropic Claude` 与 `OpenAI-compatible`。
- 每套配置只暴露 5 个用户填写字段：配置名称、Provider、Base URL、API Key、Model Id。
- API Key 写入系统 Keychain/Keystore；配置列表仅保存 `apiKeySecretRef`、Provider、Base URL、Model Id、名称和激活状态。
- 默认界面显示当前配置下拉框，旁边提供测试连接和删除按钮；右上角新增按钮展开表单。
- 支持多套配置新增、编辑、删除、切换激活。
- 连接测试调用真实供应商接口：Claude 使用 `/v1/messages`，OpenAI-compatible 使用 `/v1/chat/completions`。
- Base URL 支持填写服务根地址，也支持已包含 `/v1` 的地址。

#### F2: GitHub 集成

- 当前真实入口是手动 GitHub Token，作为 OAuth Proxy 未完成前的过渡方案。
- Token 保存到系统安全存储，前端只保存授权状态，不读取 token 明文。
- 已接入 GitHub REST / Git Data API：
  - `/user/repos` 获取真实仓库列表，支持分页加载。
  - branches API 获取目标分支并记录分支 ref。
  - tree API 读取 recursive tree，用于项目快照。
  - content/git blob 相关能力用于读取原文件和提交变更。
  - Git Data API 六步提交：get ref、get commit、get tree、create blob/tree、create commit、update ref。
- GitHub OAuth + deep link callback 仍是后续计划，不在当前已实现范围内。

#### F3: 项目快照（Project Snapshot）

- 选择项目后基于真实 GitHub recursive tree 生成项目快照。
- 快照包含：
  - 目录树
  - 技术栈与依赖摘要
  - 关键文件路径、角色与摘要
  - 模块边界
  - 被跳过文件及原因
  - 生成时间
- 文件内容读取包含大小、二进制、Git LFS 和编码保护；不适合进入上下文的文件会被记录为 skipped file。
- 快照按 repo、branch、commit/ref 维度缓存，支持手动刷新。
- 生成过程通过 Tauri event 报告进度。

#### F4: 对话式开发

- Chat UI 支持真实 AI 流式输出、停止生成、错误展示、重试和关闭错误。
- Prompt Builder 会注入当前项目、分支、项目快照、最近对话与输出格式要求。
- Prompt 与历史上下文会裁剪到预算内，并对 API key、token、password、secret 类文本做脱敏。
- AI 输出需要遵守 `DEVCHAT_CHANGESET` 合同：Summary、Impact、Commit Message、diff 代码块；无变更时返回 `NO_DIFF: 原因`。
- AI 回复中的 unified diff 会被解析为 pending changes；格式错误时显示“无法解析”，不会生成假 Diff。
- 已提交会话为只读，用户需要新建会话后继续开发。

#### F5: Diff 展示与审查

- 变更页展示所有 pending diffs。
- 支持逐文件勾选/取消，选择状态会写回当前会话。
- 支持文件级折叠/展开。
- 支持修改、新增、删除、重命名文件；重命名展示 `previousPath → filePath`。
- 大 Diff 最多渲染前 400 行，并显示截断提示。
- 语法高亮、行级评论和更完整的移动端手势仍为后续增强。

#### F6: 提交与推送

- Commit 页面读取当前 selected diffs、目标分支和 Commit Message。
- 提交前会刷新目标分支 ref，发现远程分支变化时阻止提交。
- 对每个 selected diff 读取真实原文件内容并应用 patch，生成 GitHub Git Data API 所需的最终文件内容。
- Patch 上下文不匹配、二进制文件、不支持变更类型或空变更都会阻止提交并显示错误。
- 成功后返回 commit sha / URL，清理当前 pending changes，并把当前会话标记为 committed。
- 当前页面仍需补齐二次确认、默认使用 AI commit message 候选、成功后更完整的用户反馈。

#### F7: 历史会话

- 本地保存版本化 `SessionHistory`。
- 历史页按 repo 和 branch 分组展示会话。
- 未提交会话可继续；已提交会话只读并展示提交状态。
- 支持删除历史会话。
- 历史数据只保存消息、pending changes、commit 元数据和项目标识，不保存 API Key 或 GitHub Token。

#### F8: 官网 App

- 官网使用独立 Next.js App 承载产品定位、能力介绍、下载入口、更新公告和文档入口。
- 当前官网是英文主导、中文辅助的响应式展示站。
- 与主项目共享品牌方向和部分视觉语言，但不承载用户密钥、GitHub Token、项目快照或会话数据。
- 后续需要补齐真实下载渠道、SEO、sitemap、robots、OG、帮助中心和隐私友好统计。

## 3. 用户角色与权限

| 角色 | 权限 |
|------|------|
| 未配置用户 | 可进入 App、查看空态、配置 AI 与 GitHub Token |
| 已配置 AI | 可测试连接、进行对话；未选择项目时只能获得受限回答 |
| 已配置 GitHub Token | 可加载真实仓库、生成项目快照、选择分支、对话生成 Diff、Commit & Push |
| 已完成提交的会话用户 | 可回看只读会话与提交记录，不可在已提交会话中继续编辑 |

## 4. 数据模型

### 4.1 核心实体

```
UserConfig
├── version: 1
├── aiConfigs: AiConfig[]
├── githubAuthStatus: not_configured | configured | invalid | expired
└── preferences: UserPreferences

AiConfig
├── id: string
├── name: string
├── provider: claude | openai-compatible
├── baseUrl: string
├── model: string
├── apiKeySecretRef: string
└── isActive: boolean

UserPreferences
├── theme: system | light | dark
├── language: zh-CN | en-US
└── defaultBranch: string

Project
├── repoId: string
├── repoOwner: string
├── repoName: string
├── repoFullName: string
├── branch: string
├── branchSha?: string
├── snapshot?: ProjectSnapshot
└── lastAccessed: string

ProjectSnapshot
├── directoryTree: object
├── techStack: { language, framework, dependencies }
├── keyFiles: [{ path, role, summary }]
├── moduleMap: object
├── skippedFiles: [{ path, reason }]
└── generatedAt: string

SessionHistory
├── version: 1
└── sessions: Session[]

Session
├── id: string
├── projectId: string
├── repoFullName: string
├── branch: string
├── title: string
├── messages: Message[]
├── pendingChanges: FileDiff[]
├── status: active | committed
├── commitSha?: string
├── commitUrl?: string
├── createdAt: string
└── updatedAt: string

FileDiff
├── filePath: string
├── previousFilePath?: string
├── type: added | modified | deleted | renamed
├── hunks: DiffHunk[]
├── additions: number
├── deletions: number
├── rawDiff: string
└── selected: boolean

CommitResult
├── sha: string
└── htmlUrl?: string
```

### 4.2 敏感数据规则

- 普通本地 store 不保存 API Key 或 GitHub Token 明文。
- AI Key 只通过 `apiKeySecretRef` 从系统安全存储读取。
- GitHub Token 当前固定使用内部 secret ref 保存到系统安全存储。
- 前端只读取“是否配置”与授权状态，不提供 secret 明文读取能力。
- 测试、日志、任务报告和文档禁止写入真实凭据。

## 5. 非功能性需求

| 维度 | 要求 |
|------|------|
| **性能** | 项目快照生成过程有进度反馈；缓存命中应秒级进入；AI 响应使用 streaming |
| **安全** | API Key 与 GitHub Token 使用系统安全存储；Prompt、历史和日志需要脱敏 |
| **可靠性** | 外部 API 错误需要转为用户可读信息；提交前必须校验分支 ref 与 patch 上下文 |
| **离线** | 最近项目、项目快照和历史会话可离线查看 |
| **可维护性** | AI、GitHub、storage 通过 Tauri command 和 service 分层，外部 API 通过 fake server/fixture 测试 |
| **官网体验** | 首屏快速加载，移动端与桌面端都能清晰完成产品理解、下载和文档访问 |
