# DevChat 真实功能实现计划

## Objective

将 `apps/devchat` 从 MVP demo adapter 逐步替换为真实功能链路。第一阶段优先打通 GitHub 项目上下文：用户在设置页保存 GitHub token 后，仓库列表、项目选择和项目快照从真实 GitHub REST API 获取数据，不再依赖前端 mock 初始数据。

## Assumptions

- 本计划针对主项目 App：`apps/devchat`。
- 在 GitHub OAuth Proxy 完成前，先用用户手动保存的 GitHub PAT 作为真实凭据入口。
- 前端只持有 `tokenSecretRef`，不保存、不读取 GitHub token 明文。
- 浏览器独立预览不能提供完整真实能力；真实 GitHub / AI / commit 能力以 Tauri 运行时为准。

## Commands

- Build: `pnpm --filter devchat build`
- Test: `pnpm --filter devchat test`
- Lint: `pnpm --filter devchat lint`
- Rust tests: `cd apps/devchat/src-tauri && cargo test`

## Project Structure

- `apps/devchat/src-tauri/src/services/`：真实外部 API client 和纯逻辑服务。
- `apps/devchat/src-tauri/src/commands/`：Tauri command 边界，负责读取 secret ref、校验输入、调用 service。
- `apps/devchat/src/stores/`：前端状态，只保存非敏感状态和 secret ref。
- `apps/devchat/src/pages/`：真实功能所需的设置、加载、错误和空状态。
- `docs/任务报告.md`：每次完成明确切片后追加验证记录。

## Testing Strategy

- Rust service 使用本地 TCP test server 覆盖 GitHub REST request/response 映射，不触网。
- 前端 store 使用 Vitest mock Tauri invoke，验证参数契约和状态流。
- 外部服务调用测试不得写入真实 token；真实手动验证只在本机配置中进行。

## Boundaries

- Always: secret 只经 Rust command 读取；外部响应按结构化类型解析；每个切片运行相关测试。
- Ask first: OAuth Proxy 位置、真实 GitHub token 权限范围、引入新依赖、改动 CI。
- Never: 提交 `.env`、API Key、GitHub token、个人机器凭据；前端 localStorage 保存认证 token。

## Success Criteria For Current Slice

- 设置页能保存 GitHub token 到 Rust store。
- 项目页加载仓库时调用 `github_list_repos`，只传 `tokenSecretRef`。
- Rust `GitHubClient` 调用真实 GitHub REST API，并通过本地 mock server 测试。
- 选择项目时通过真实 tree 生成快照。
- `pnpm --filter devchat test`、`pnpm --filter devchat lint`、`cd apps/devchat/src-tauri && cargo test` 通过。
