# DevChat — 交互设计文档

> 当前同步日期：2026-05-12
> 本文档以当前主项目 App 与官网 App 的实现为准，并标注后续计划能力。

## 1. 信息架构

DevChat 包含两个用户界面：主项目 App 面向对话式代码修改与提交，官网 App 面向产品展示、下载转化和文档入口。

### 1.1 主项目 App

```
App Shell
├── 首页（当前/最近项目 + 快捷入口）
├── 项目（GitHub Repo 列表）
├── 变更（Pending Diff 审查）
├── 历史（会话记录）
└── 设置（AI 配置 / GitHub Token / 偏好）

流程页面
├── 对话页 /chat/:projectId
├── 提交确认页 /commit
└── 兜底路由返回首页
```

底部导航当前为 5 项：`首页`、`项目`、`变更`、`历史`、`设置`。对话页从首页当前项目、项目列表或历史会话进入。

### 1.2 官网 App

```
Website
├── 首页（产品矩阵 + 关键能力 + CTA）
├── 功能（对话开发 / Diff 审查 / 安全边界 / GitHub 工作流）
├── 下载（平台状态与下载入口）
├── 更新公告
└── 文档与支持
```

## 2. 页面流程

### 2.1 主项目 App 首次使用流程

```
启动 App
  → 进入首页
  → 进入设置
     ├── AI 模型配置
     │   ├── 默认展示当前配置下拉框
     │   ├── 点击右上角新增，展开表单
     │   ├── 填写配置名称、Provider、Base URL、API Key、Model Id
     │   ├── 保存配置：API Key 写入系统安全存储，配置元数据写入本地 store
     │   └── 点击测试连接：请求真实供应商接口
     └── GitHub
         ├── 输入 GitHub Token
         └── 保存到系统安全存储
  → 进入项目页
  → 加载真实 GitHub 仓库
```

说明：GitHub OAuth 网页授权和 deep link callback 尚未接入；当前交互以 GitHub Token 作为真实功能过渡入口。

### 2.2 主项目 App 核心开发流程

```
选择项目
  → 读取 repo 默认分支
  → 生成或加载项目快照
     ├── 首次：显示快照进度
     └── 缓存命中：直接进入对话
  → 进入对话页
  → 用户输入需求
  → AI 流式返回响应
     ├── 文本说明：实时渲染 Markdown
     ├── 合规 Diff：解析为 pending changes
     └── 格式错误：显示无法解析，不生成假 Diff
  → 用户进入变更页审查
     ├── 勾选/取消文件
     ├── 折叠/展开文件 Diff
     └── 大 Diff 截断展示
  → 进入提交确认页
     ├── 选择/确认目标分支
     ├── 编辑 Commit Message
     ├── 提交前刷新远程分支 ref
     ├── 读取原文件内容并应用 patch
     └── GitHub Git Data API Commit & Push
  → 提交成功
     ├── 显示 commit sha
     ├── 清理 pending changes
     └── 当前会话标记为已提交只读
```

### 2.3 历史会话流程

```
进入历史页
  → 按 repo / branch 分组展示会话
  → 点击未提交会话
     └── 回到对话页继续
  → 点击已提交会话
     └── 回到只读对话页
  → 点击删除
     └── 删除本地会话记录
```

### 2.4 官网访问流程

```
进入官网首页
  → 查看产品矩阵与关键能力
  → 点击下载入口
     └── 查看平台下载状态
  → 需要了解更多
     ├── 查看功能详情
     ├── 查看更新公告
     └── 进入文档与支持
```

### 3.3 对话页

```
┌─────────────────────────────┐
│ repo · main        [+][史][刷][变] │
│ React / Tauri                │
├─────────────────────────────┤

│ 分支 main                    │

├─────────────────────────────┤

├─────────────────────────────┤
│ AI / 用户消息流              │
│ ┌───────────────────────┐   │
│ │ AI Markdown 响应       │   │
│ └───────────────────────┘   │
│ ┌───────────────────────┐   │
│ │ 查看全部变更：2 个文件 │   │

├─────────────────────────────┤
│ 输入你的需求...        [发送]│
└─────────────────────────────┘
```

顶部操作：
- `+`：开始新会话。
- `历史`：进入历史会话。
- `刷新`：手动刷新当前项目快照。
- `变更`：进入 Diff 审查页。

状态规则：
- 未选择项目时标题显示“未选择项目”，上下文条显示等待快照。
- AI streaming 中，空 assistant message 显示“正在生成...”。
- 生成中发送按钮变为停止按钮。
- 已提交会话输入框禁用，placeholder 显示“已提交会话不可继续编辑”。
- 错误区显示错误来源，并提供重试和关闭按钮。

### 3.4 变更审查页

```
┌─────────────────────────────┐
│ ← 变更详情                  │

├─────────────────────────────┤

├─────────────────────────────┤
│ ☑ src/App.tsx      +18 -3  │
│ ☑ src/App.css      +12 -0  │
├─────────────────────────────┤
│ ▾ src/App.tsx              │
│ ┌─────────────────────────┐ │
│ │ @@ -1,5 +1,9 @@         │ │
│ │ - old line              │ │
│ │ + new line              │ │
│ └─────────────────────────┘ │
├─────────────────────────────┤
│ [提交变更]                  │
└─────────────────────────────┘
```

交互规则：
- 点击文件项切换 selected 状态。
- 点击文件标题前的箭头折叠/展开 Diff。
- 重命名文件显示为 `oldPath → newPath`。
- 每个文件最多渲染前 400 行，超过时显示截断提示。
- 点击“提交变更”进入提交确认页。

### 3.5 提交确认页

```
┌─────────────────────────────┐
│ 提交确认                    │
│ 选择文件，编辑 message。     │
├─────────────────────────────┤
│ 目标分支 main           ▾   │
├─────────────────────────────┤
│ Commit Message              │
│ ┌───────────────────────┐  │
│ │ feat: add search       │  │
│ └───────────────────────┘  │
├─────────────────────────────┤
│ 修改文件                    │
│ ☑ src/App.tsx (+18-3)      │
│ ☑ src/App.css (+12-0)      │
├─────────────────────────────┤
│ [Commit & Push]             │
│ 提交成功 abc123             │
└─────────────────────────────┘
```

当前行为：
- Commit Message 可编辑，当前默认文案仍是页面默认值。
- 提交前会刷新远程分支 ref。
- 没有项目、没有选中文件、分支已变化、patch apply 失败或 GitHub API 失败都会显示状态文本。
- 成功后显示 commit sha，并将当前会话标记为 committed。

后续增强：
- 默认填充 AI 输出中的 Commit Message 候选。
- Push 前二次确认。
- 成功态展示 commit 链接、文件统计和下一步入口。

## 4. 交互细节

### 4.1 对话交互

| 交互 | 当前行为 |
|------|----------|
| 发送消息 | 追加用户消息，启动真实 AI streaming，逐 chunk 更新 assistant message |
| 停止生成 | 前端停止当前生成状态并忽略旧 stream 后续事件 |
| 失败重试 | 使用最后失败的用户消息重新发起 streaming |
| 关闭错误 | 清除当前错误状态 |
| 查看变更 | pending diffs 存在时显示“查看全部变更”卡片 |
| 已提交会话 | 输入禁用，阻止继续发送 |

计划中但未实现：
- 长按消息复制 / 重新生成 / 删除。
- 输入 `@` 触发文件选择器。
- 上滑加载更早历史。

### 4.2 Diff 交互

| 交互 | 当前行为 |
|------|----------|
| 点击文件 | 切换该文件是否参与提交 |
| 折叠文件 | 折叠/展开单个文件 Diff |
| 大 Diff | 每个文件最多显示前 400 行 |
| Rename | 同时展示旧路径和新路径 |
| 横向滚动 | Diff 内容保留代码格式，窄屏可横向查看 |

计划中但未实现：
- 点击行号展开上下文。
- 语法高亮。
- 双指缩放调整字号。

### 4.3 提交交互

| 交互 | 当前行为 |
|------|----------|
| Commit Message | 用户可编辑，当前默认值来自页面默认文案 |
| 目标分支 | 使用真实分支列表；提交前刷新远程 ref |
| Commit & Push | 应用 patch 后调用 GitHub Git Data API |
| 提交成功 | 显示 commit sha，清空 pending changes，标记会话 committed |
| 提交失败 | 显示错误原因，用户可修正后再次点击提交 |

计划中但未实现：
- Push 前二次确认。
- 默认使用 AI Commit Message 候选。
- 成功态展示 commit URL 和更完整的下一步操作。

### 4.4 设置交互

| 交互 | 当前行为 |
|------|----------|
| AI 配置下拉 | 切换并激活当前配置 |
| 新增 AI 配置 | 创建草稿配置并展开表单 |
| 保存 AI 配置 | API Key 入系统安全存储，配置元数据入本地 store |
| 测试连接 | 使用当前激活配置请求真实 AI provider |
| 删除配置 | 删除当前配置；至少保留一套配置 |
| 保存 GitHub Token | Token 入系统安全存储，刷新 GitHub 授权状态 |

## 6. 状态与反馈

### 6.1 加载状态

| 场景 | 反馈方式 |
|------|----------|
| Repo 列表加载 | 文案“正在加载 GitHub 仓库...” |
| 项目快照生成中 | 快照进度文案 + 百分比 |
| AI 回复中 | 流式文字；空 assistant message 显示“正在生成...” |
| Push 中 | 状态文本“正在提交...” |
| 历史加载失败 | 页面顶部错误文案 |

### 6.2 空状态

| 场景 | 展示 |
|------|------|
| 无最近项目 | “还没有最近项目” |
| 无 GitHub Token | “保存 GitHub Token 后加载仓库” |
| 搜索无结果 | “没有匹配的仓库”或“没有匹配的最近项目” |
| 无历史会话 | “暂无历史会话” |
| 未选择项目 | 对话页显示“未选择项目”和“等待快照” |
| 无 pending diff | 变更页仅显示空文件列表与提交入口，后续需补更明确空态 |

### 6.3 错误处理

| 错误类型 | 处理方式 |
|----------|----------|
| AI 连接失败 | 设置页状态显示连接失败，并展示真实错误 |
| AI 输出无法解析 | 对话页显示“无法解析 AI 输出的 diff” |
| GitHub Token 缺失或无效 | 项目页显示仓库加载失败，引导保存 Token |
| 分支已变化 | 提交页提示远程分支已变化，阻止提交 |
| Patch apply 失败 | 提交页显示上下文不匹配或文件不支持的具体原因 |
| GitHub API 失败 | 提交页展示 Rust command 返回的可读错误 |
| Tauri 运行时缺失 | 设置页保存 secret 失败时提示确认当前运行在 Tauri App 中 |

## 7. 设计规范

### 7.1 主项目视觉

| 元素 | 当前规范 |
|------|----------|
| 背景 | `#f5f5f7` |
| 主文本 | `#1d1d1f` |
| 首页 Hero | 黑色背景 + 蓝色强调 |
| 卡片 | 白色卡片，8px radius，轻量阴影 |
| 底部导航 | 深色半透明 glass，5 项固定导航 |
| 交互强调 | 蓝色链接/状态与 lucide 图标 |
| 代码展示 | `<pre>` 保留等宽格式与横向滚动 |

### 7.2 字体

| 用途 | 字体 | 大小 |
|------|------|------|
| 界面文字 | SF Pro Text / Helvetica Neue / Arial / sans-serif | 14-16px |
| 标题 | SF Pro Text / Helvetica Neue / Arial / sans-serif | 页面内按容器缩放 |
| 代码展示 | 浏览器等宽字体 | 12-14px |

### 7.3 控件原则

- 主要动作使用按钮或 icon button，并提供 `aria-label`。
