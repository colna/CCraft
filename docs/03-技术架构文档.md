# DevChat — 技术架构文档

## 1. 技术栈总览

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **仓库架构** | Turborepo + pnpm workspace | 同时管理主项目 App、官网 App 和共享包 |
| **主项目 App** | Tauri 2.0 | Rust 核心 + 系统 WebView，支持 iOS/Android/Desktop |
| **主项目前端** | React 18 + TypeScript | 组件化 UI，生态丰富 |
| **官网 App** | Next.js App Router + TypeScript | 产品官网、下载入口、文档入口、SEO |
| **状态管理** | Zustand | 主项目 App 内轻量状态管理 |
| **主项目路由** | React Router 7 | Tauri WebView 内 SPA 路由 |
| **主项目构建** | Vite 6 | 极速 HMR，Tauri 官方推荐 |
| **官网构建** | Next.js | SSR/SSG、Metadata、Image/Font 优化 |
| **样式** | Tailwind CSS 4 | 主项目 App 与官网 App 共享设计 token |
| **网络请求** | Rust reqwest (后端) + fetch (前端) | Rust 侧处理敏感请求，前端处理流式 SSE |
| **本地存储** | tauri-plugin-store + Keychain/Keystore | 配置存储 + 敏感数据加密 |
| **Git 操作** | GitHub REST API v3 | 无需本地 git，纯 API 操作 |
| **代码高亮** | Shiki | VS Code 同款语法高亮引擎 |
| **Markdown** | react-markdown + remark-gfm | 对话内容渲染 |
| **Diff 渲染** | react-diff-viewer-continued | 成熟的 Diff 展示组件 |

## 2. 为什么选 Turborepo + Tauri + Next.js

项目同时包含一个安全敏感的主项目 App 和一个面向公开访问的官网 App，因此使用 Turborepo 管理多应用和共享包：

- 主项目 App 独立放在 `apps/devchat`，聚焦 Tauri、Rust 安全层和移动端体验。
- 官网 App 独立放在 `apps/web`，使用 Next.js + Tailwind 承载 SEO、下载转化、公告和文档入口。
- 共享代码放在 `packages/*`，避免复制品牌 token、基础 UI、类型定义和 lint/tsconfig 配置。
- 根目录统一执行 `pnpm dev`、`pnpm build`、`pnpm test`、`pnpm lint`，由 Turborepo 负责任务编排和缓存。

### 2.1 为什么选 Tauri 2.0

| 维度 | Tauri 2.0 | Flutter | React Native |
|------|-----------|---------|--------------|
| 包体积 | ~5MB | ~15MB | ~10MB |
| 内存占用 | 低（系统 WebView） | 中（Skia 引擎） | 中（JS 引擎） |
| 移动端支持 | iOS + Android（2.0 新增） | 成熟 | 成熟 |
| 安全性 | Rust 内存安全 + 权限沙箱 | 一般 | 一般 |
| Web 生态 | 完整复用 npm 生态 | Dart 生态较小 | JS 生态 |
| 敏感操作 | Rust 后端处理，不暴露给 WebView | 全在 Dart | Bridge 调用 |
| 桌面扩展 | 天然支持 macOS/Windows/Linux | 需额外适配 | 不支持 |

**核心优势：** Rust 侧处理 API Key 加密存储、GitHub token 管理等敏感操作，WebView 前端完全不接触密钥明文。

### 2.2 为什么官网选 Next.js + Tailwind

| 维度 | Next.js + Tailwind | 说明 |
|------|--------------------|------|
| SEO | Metadata API + 静态生成 | 首页、下载页、更新公告适合被搜索和分享 |
| 性能 | App Router + Server Components | 默认减少客户端 JS，适合内容型官网 |
| 资产优化 | `next/image` + `next/font` | 优化产品截图、品牌图和字体加载 |
| 样式一致性 | Tailwind 共享配置 | 与主项目 App 复用色彩、间距和组件 token |
| 部署 | Vercel 或 Node/standalone | 可独立于移动端发版节奏上线 |

### 5.1 Rust 后端 — Tauri Commands

#### AI 模块

```rust
// apps/devchat/src-tauri/src/commands/ai.rs

/// 测试 AI 连接
#[tauri::command]
async fn test_ai_connection(
    base_url: String,
    api_key: String,
    model: String,
) -> Result<bool, String> {
    let client = AiClient::new(&base_url, &api_key, &model);
    client.test_connection().await.map_err(|e| e.to_string())
}

/// 流式对话 — 通过 Tauri Event 推送给前端
#[tauri::command]
async fn chat_stream(
    app: AppHandle,
    base_url: String,
    api_key: String,
    model: String,
    messages: Vec<ChatMessage>,
    system_prompt: String,
) -> Result<(), String> {
    let client = AiClient::new(&base_url, &api_key, &model);

    let mut stream = client
        .chat_stream(&messages, &system_prompt)
        .await
        .map_err(|e| e.to_string())?;

    while let Some(chunk) = stream.next().await {
        match chunk {
            Ok(text) => {
                app.emit("ai-stream-chunk", &text).unwrap();
            }
            Err(e) => {
                app.emit("ai-stream-error", &e.to_string()).unwrap();
                break;
            }
        }
    }

    app.emit("ai-stream-done", ()).unwrap();
    Ok(())
}
```

#[tauri::command]
async fn save_secret(
    app: AppHandle,
    key: String,
    value: String,
) -> Result<(), String> {

    // 使用平台原生安全存储

    app: AppHandle,

    use tauri_plugin_store::StoreExt;
    let entry = app.store("secrets.json")
        .map_err(|e| e.to_string())?;
    Ok(entry.get(&key).map(|v| v.as_str().unwrap_or("").to_string()))

### 5.2 React 前端

#### Zustand 状态管理

```typescript
// apps/devchat/src/stores/chatStore.ts

interface ChatState {
  messages: Message[];
  isGenerating: boolean;
  pendingDiffs: FileDiff[];

  sendMessage: (content: string) => Promise<void>;
  stopGeneration: () => void;
  clearMessages: () => void;
}

export const useChatStore = create<ChatState>((set, get) => ({
  messages: [],
  isGenerating: false,
  pendingDiffs: [],

  sendMessage: async (content: string) => {
    const userMsg: Message = { role: 'user', content };
    set(state => ({
      messages: [...state.messages, userMsg],
      isGenerating: true,
    }));

    const aiMsg: Message = { role: 'assistant', content: '' };
    set(state => ({
      messages: [...state.messages, aiMsg],
    }));

    // 监听 Rust 侧推送的流式数据
    const unlisten = await listen<string>('ai-stream-chunk', (event) => {
      set(state => {
        const msgs = [...state.messages];
        const last = msgs[msgs.length - 1];
        last.content += event.payload;
        return { messages: msgs };
      });
    });

    const unlistenDone = await listen('ai-stream-done', () => {
      const fullResponse = get().messages[get().messages.length - 1].content;
      const diffs = parseDiffsFromResponse(fullResponse);
      set({ isGenerating: false, pendingDiffs: diffs });
      unlisten();
      unlistenDone();
    });

    // 调用 Rust 命令启动流式对话
    const config = useAIConfigStore.getState().activeConfig;
    const snapshot = useProjectStore.getState().currentSnapshot;
    await invoke('chat_stream', {
      baseUrl: config.baseUrl,
      apiKey: config.apiKey,
      model: config.model,
      messages: get().messages.map(m => ({ role: m.role, content: m.content })),
      systemPrompt: buildSystemPrompt(snapshot),
    });
  },

  stopGeneration: () => set({ isGenerating: false }),
  clearMessages: () => set({ messages: [], pendingDiffs: [] }),
}));
```

// apps/devchat/src/hooks/useAI.ts

export function useAI() {
  const { messages, isGenerating, pendingDiffs, sendMessage, stopGeneration } =
    useChatStore();

  const send = useCallback(async (content: string) => {
    try {
      await sendMessage(content);
    } catch (err) {
      console.error('AI chat error:', err);
    }
  }, [sendMessage]);

  return { messages, isGenerating, pendingDiffs, send, stopGeneration };
}

#### Diff 解析器（前端侧）

```typescript
// apps/devchat/src/lib/diffParser.ts

export interface FileDiff {
  filePath: string;
  type: 'added' | 'modified' | 'deleted';
  hunks: DiffHunk[];
  additions: number;
  deletions: number;
  rawDiff: string;
}

export function parseDiffsFromResponse(response: string): FileDiff[] {
  const diffs: FileDiff[] = [];
  const pattern = /```diff\n([\s\S]*?)```/g;
  let match: RegExpExecArray | null;

  while ((match = pattern.exec(response)) !== null) {
    const diffContent = match[1];
    const fileDiff = parseSingleDiff(diffContent);
    if (fileDiff) diffs.push(fileDiff);
  }

  return diffs;
}

function parseSingleDiff(content: string): FileDiff | null {
  const lines = content.split('\n');
  const filePathMatch = lines[0]?.match(/^--- a\/(.+)/);
  if (!filePathMatch) return null;

  const filePath = filePathMatch[1];
  let additions = 0;
  let deletions = 0;

  for (const line of lines) {
    if (line.startsWith('+') && !line.startsWith('+++')) additions++;
    if (line.startsWith('-') && !line.startsWith('---')) deletions++;
  }

  return {
    filePath,
    type: additions > 0 && deletions > 0 ? 'modified' : additions > 0 ? 'added' : 'deleted',
    hunks: parseHunks(lines),
    additions,
    deletions,
    rawDiff: content,
  };
}
```

## 6. Tauri IPC 通信设计

前端与 Rust 后端通过 Tauri IPC 通信，分为两种模式：

### 6.1 Command 模式（请求-响应）

```
前端 invoke('command_name', args)  →  Rust #[tauri::command]  →  返回 Result
```

| 命令 | 用途 |
|------|------|
| `test_ai_connection` | 测试 AI API 连接 |
| `github_list_repos` | 获取仓库列表 |
| `github_get_tree` | 获取文件树 |
| `github_get_file` | 获取文件内容 |
| `github_commit_and_push` | 提交推送 |
| `generate_snapshot` | 生成项目快照 |
| `save_secret` / `get_secret` | 安全存储读写 |

### 6.2 Event 模式（流式推送）

```
Rust app.emit("event_name", payload)  →  前端 listen("event_name", callback)
```

| 事件 | 用途 |
|------|------|
| `ai-stream-chunk` | AI 流式响应的每个 token |
| `ai-stream-done` | AI 响应完成 |
| `ai-stream-error` | AI 响应出错 |
| `snapshot-progress` | 快照生成进度 |

### 8.2 敏感数据隔离

| 数据 | 存储位置 | 访问方式 |
|------|----------|----------|
| API Key | Rust 侧 tauri-plugin-store（加密） | 仅通过 Rust Command 使用，前端不接触明文 |
| GitHub Token | Rust 侧安全存储 | 同上 |
| 项目快照 | tauri-plugin-store | 前端可读 |
| 对话历史 | tauri-plugin-store | 前端可读写 |
| 用户偏好 | localStorage | 前端直接读写 |

**关键原则：** API Key 和 Token 只存在于 Rust 进程中，前端 WebView 永远不会接触到明文密钥。所有涉及密钥的网络请求都在 Rust 侧发起。

### 8.3 网络安全

- 所有请求强制 HTTPS（Tauri Capabilities 限制域名）
- GitHub OAuth 使用 PKCE 流程
- 请求超时：连接 10s，AI 响应 180s
- Rust 侧日志不输出敏感字段

## 10. 关键 Rust 实现

### 10.1 AI 流式客户端

```rust
// apps/devchat/src-tauri/src/services/ai_client.rs

use futures_util::StreamExt;
use reqwest::Client;
use serde_json::Value;

pub struct AiClient {
    client: Client,
    base_url: String,
    api_key: String,
    model: String,
}

impl AiClient {
    pub async fn chat_stream(
        &self,
        messages: &[ChatMessage],
        system_prompt: &str,
    ) -> Result<impl futures_util::Stream<Item = Result<String, anyhow::Error>>, anyhow::Error> {
        let mut all_messages = vec![ChatMessage {
            role: "system".into(),
            content: system_prompt.into(),
        }];
        all_messages.extend_from_slice(messages);

        let response = self.client
            .post(format!("{}/chat/completions", self.base_url))
            .header("Authorization", format!("Bearer {}", self.api_key))
            .json(&serde_json::json!({
                "model": self.model,
                "messages": all_messages,
                "stream": true,
            }))
            .send()
            .await?;

        let stream = response.bytes_stream().map(|chunk| {
            let bytes = chunk?;
            let text = String::from_utf8_lossy(&bytes);
            let mut result = String::new();

            for line in text.lines() {
                if let Some(data) = line.strip_prefix("data: ") {
                    if data == "[DONE]" { continue; }
                    if let Ok(json) = serde_json::from_str::<Value>(data) {
                        if let Some(content) = json["choices"][0]["delta"]["content"].as_str() {
                            result.push_str(content);
                        }
                    }
                }
            }

            Ok(result)
        });

        Ok(stream)
    }
}
```

### 10.2 GitHub Git Data API 提交

```rust
// apps/devchat/src-tauri/src/services/github_client.rs

impl GitHubClient {
    pub async fn commit_and_push(
        &self,
        owner: &str,
        repo: &str,
        branch: &str,
        changes: &[FileChange],
        message: &str,
    ) -> Result<CommitResult, anyhow::Error> {
        // 1. 获取当前分支的最新 commit SHA
        let ref_data: Value = self.get(
            &format!("/repos/{owner}/{repo}/git/ref/heads/{branch}")
        ).await?;
        let latest_sha = ref_data["object"]["sha"].as_str().unwrap();

        // 2. 获取 commit 对应的 tree SHA
        let commit_data: Value = self.get(
            &format!("/repos/{owner}/{repo}/git/commits/{latest_sha}")
        ).await?;
        let base_tree_sha = commit_data["tree"]["sha"].as_str().unwrap();

        // 3. 为每个修改文件创建 blob
        let mut tree_items = Vec::new();
        for change in changes {
            match change.change_type {
                ChangeType::Delete => {
                    tree_items.push(serde_json::json!({
                        "path": change.path,
                        "mode": "100644",
                        "type": "blob",
                        "sha": null
                    }));
                }
                _ => {
                    let blob: Value = self.post(
                        &format!("/repos/{owner}/{repo}/git/blobs"),
                        &serde_json::json!({
                            "content": change.content,
                            "encoding": "utf-8"
                        }),
                    ).await?;
                    tree_items.push(serde_json::json!({
                        "path": change.path,
                        "mode": "100644",
                        "type": "blob",
                        "sha": blob["sha"]
                    }));
                }
            }
        }

        // 4. 创建新 tree
        let new_tree: Value = self.post(
            &format!("/repos/{owner}/{repo}/git/trees"),
            &serde_json::json!({
                "base_tree": base_tree_sha,
                "tree": tree_items
            }),
        ).await?;

        // 5. 创建 commit
        let new_commit: Value = self.post(
            &format!("/repos/{owner}/{repo}/git/commits"),
            &serde_json::json!({
                "message": message,
                "tree": new_tree["sha"],
                "parents": [latest_sha]
            }),
        ).await?;

        // 6. 更新分支引用 = push
        self.patch(
            &format!("/repos/{owner}/{repo}/git/refs/heads/{branch}"),
            &serde_json::json!({
                "sha": new_commit["sha"]
            }),
        ).await?;

        Ok(CommitResult {
            sha: new_commit["sha"].as_str().unwrap().to_string(),
        })
    }
}
```

主项目 App 保持**纯客户端架构 + Rust 安全层**。相比 Flutter 方案，Tauri 额外提供：
- Rust 内存安全保障
- API Key 物理隔离（前端 WebView 无法访问）
- Capabilities 权限白名单（精确控制网络访问范围）
- 天然支持未来扩展到桌面端

官网 App 与主项目 App 独立部署、独立发布；它只暴露公开内容和下载入口，不处理用户 API Key、GitHub Token 或项目快照。
